# Design

本文档记录当前仓库实际成立的产品方向、实现状态和约束。稳定需求与验收状态见 [[REQUIREMENTS]]，阶段记录见 [[PROGRESS]]。

## Product Direction

《调查》系列当前包含多个游戏项目：

- 《调查深入》桌游：现有规则和 CSV 数据的主要来源。
- 《调查深入》电子游戏：当前主线，目标是完成一阶段 TypeScript 版本，让桌游玩法先以电子化方式跑通。
- 《现实解构》：另一个桌游/DLC 方向，当前作为独立资料区和未来扩展参考。

当前优先级：

1. 将 [[调查深入-GDD]] 的一阶段收敛为可玩的 TypeScript MVP。
2. 保持玩法与 [[调查深入-规则指引书]]、[[调查附录]]、[[FAQ]]、[[调查深入-卡牌统计表]] 可追溯。
3. 先跑通单机原型与规则结算，再逐步拆分 P2P、多指挥者和更完整的策略/环境 UI。

目标用户：

- 设计者：快速验证桌游规则电子化后的节奏和状态复杂度。
- 后续 agent/开发者：基于稳定资料源继续拆解系统、实现玩法、补测试。
- 玩家验证者：通过浏览器原型进行基础流程试玩。

## Current Implementation

- Stack: Vite + TypeScript + GSAP。
- Language: TypeScript。
- UI framework: 原生 DOM 渲染，没有 React/Vue。
- Core libraries: GSAP 用于入场、发牌、数值反馈等动画。
- Data model: 从 `investigation-delve/boardgame/game-design/调查深入-卡牌统计表/*.csv` 直接加载调查员、策略、环境、辅助和情报数据。
- Persistence: 暂无持久化；当前状态存在浏览器内存。
- Implementation registry: `investigation-delve/videogame/ts/src/data/implementationRegistry.json` 记录 CSV 记录与可执行 hook 的映射状态。

关键代码目录：

- `investigation-delve/videogame/ts/src/data/`: CSV 解析、资产引用、实现状态登记。
- `investigation-delve/videogame/ts/src/engine/`: 回合状态、调查、策略准备与结算。
- `investigation-delve/videogame/ts/src/ui/`: DOM 渲染、交互绑定、GSAP 动画。

## Layout / UX

Delve TS 原型当前是单屏桌面优先的战斗/调查视图：

- 左侧显示玩家队伍、调查次数选择、剩余情报。
- 中央显示情报进度、行动顺序、主场景状态消息、策略手牌。
- 右侧显示对手队伍、当前环境、策略牌堆/弃牌堆、行动记录。
- 策略牌点击后进入目标选择；可被选中的调查员卡片高亮。
- 环境卡与行动记录可展开。
- `window.render_game_to_text()` 提供文本化 QA 状态，供浏览器自动化检查。

当前 UI 仍是原型形态，重点是规则可观察、流程可操作，而不是最终美术品质。

## Architecture Decisions

- 使用 CSV 作为规则数据源：避免复制卡牌和调查员内容，保证电子化实现能回链到桌游资料。
- 使用 `implementationRegistry.json` 标注实现状态：区分已加载、可追踪折算、完整自动化和待实现效果。
- 先做单机共享状态：P2P、多指挥者手牌与私密信息会显著改变状态边界，暂不抢先抽象。
- 使用原生 DOM：当前页面较集中，原生渲染足以降低工程复杂度。
- 将复杂卡牌先做可追踪折算：优先让一阶段流程跑通，并在 [[REQUIREMENTS]] 中保留完整 UI 化任务。

## Constraints
