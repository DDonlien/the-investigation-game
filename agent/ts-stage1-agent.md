# TS Stage1 Agent Brief

这个文档服务于后续参与《调查深入》电子游戏一阶段实现的 agent。

## 目标

完成《调查深入》电子游戏的一阶段版本：

- 单人
- P2P 联机
- 核心玩法与桌游基本一致
- 允许有限的电子化扩充，但不改变核心规则身份

参考策划：

- [[调查深入-GDD]]

## 一阶段的理解

根据 [[调查深入-GDD]]，一阶段更接近：

- 原版桌游的数字化
- 核心循环先跑通
- 输入与镜头相对简单
- 不追求二阶段、三阶段的 DBG 外循环与复杂扩展

## 一阶段优先级

### 必做

- 调查员与指挥者的基础数据结构
- 卡组与卡区表示
- 基础局外流程
- 基础局内流程
- 调查 / 操作 / SAN / 疯狂等核心规则映射
- 单机完整可玩

### 高优先级

- P2P 联机的最小闭环
- 卡牌效果执行框架
- 回合结算顺序
- 日志与回放友好的状态表示

### 暂缓

- 二阶段 DBG 外循环
- 三阶段复杂化系统
- 过重的演出和美术系统
- 非必要的内容扩充

## 推荐实现顺序

1. 定义领域模型
2. 定义局外 / 局内状态机
3. 跑通无 UI 的核心规则模拟
4. 补最小 UI
5. 再接入 P2P

## 建议的数据拆分

建议优先把现有资料拆成以下实现对象：

- 调查员
- 指挥者
- 情报卡
- 策略卡
- 环境卡
- 辅助卡
- 卡组 / 手牌 / 弃牌区 / 情报区
- SAN / 疯狂状态
- 回合阶段
- 结算事件

## 对实现 agent 的提醒

- 不要一开始就直接做全量内容
- 先做“规则引擎最小闭环”
- 先确保状态可序列化，方便联机与存档
- 对模糊规则优先记录在 [[progress]]，不要默默拍脑袋扩展
